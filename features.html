Alpha v0.1.5 · Feature Overview

Capabilities built for trustworthy daily use

This release is not a flashy promise build. It is a stability + UX baseline focused on queue reliability, approval workflows, deterministic planning paths, operator visibility, and artifact hygiene.

Core capabilities

What works in Alpha v0.1.5

Mission Planning

  • voxera missions plan "<goal>" uses cloud planning while preserving local policy and approval gating.
  • Deterministic fast-path for common note/file writes emits exactly one files.write_text step.
  • Planner rewrites sandbox-inappropriate steps to safe clipboard confirmation actions when intent is not explicit.
  • Built-in, repo, and user mission files with deterministic resolution order.

Queue & Inbox Operations

  • Queue daemon processes mission/goal JSON jobs via a strict inbox intake contract.
  • Approval artifacts written to pending/approvals/*.approval.json with policy reason and scope metadata.
  • Full lifecycle: status, approvals list/approve/deny, cancel, retry, pause, resume, unlock.
  • Human-friendly queueing: voxera inbox add and voxera inbox list.

Artifact Hygiene & Retention

  • Per-job bundles under notes/queue/artifacts/<job_id>/ with plan, action timeline, stdout/stderr, and generated file metadata.
  • voxera artifacts prune with configurable --max-age-days / --max-count and dry-run by default.
  • Failed-job sidecar contract with schema versioning and deterministic validation.
  • Retention policies by age and count with prune summaries and health counters.

Services, Panel & Updates

  • Systemd user service lifecycle via Make targets: services-install, status, restart, stop, disable.
  • Panel supports queue + mission creation and insights. POST mutation endpoints by default.
  • make update / scripts/update.sh pulls, reinstalls, and restarts services safely.
  • Update flow never deletes anything under ~/VoxeraOS/notes.

Safety model

Guardrails by design

Fully audited actions

Every action records what happened, why it happened, and how to undo it where possible. Nothing is silent.

Skill-declared permissions

Skills declare their scope and network needs. Policies decide allow, ask, or deny at runtime — deterministically.

Sandbox isolation

Rootless Podman backend, read-only root filesystem, writable /work only, and network disabled by default.

Config contract

Deterministic, inspectable configuration

Strict precedence order

CLI overrides → VOXERA_* env → config file → defaults. Always predictable, never surprising.

voxera config show      # redacted view
voxera config validate  # non-zero on errors
voxera config snapshot  # write audit snapshot

Runtime config file

Optional override at ~/.config/voxera/config.json. Keep secrets out of git.

{
  "panel_host": "127.0.0.1",
  "panel_port": 8844,
  "queue_lock_stale_s": 3600,
  "panel_csrf_enabled": true
}
