Quick Start

Get Voxera OS Alpha v0.1.4 running

Choose the Alpha baseline flow or the full dev VM setup. Commands below are copyable and organized for practical operator use.

Path A

Alpha baseline quick start

python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -e ".[dev]"

make update
make services-install

voxera --version
voxera queue status
voxera inbox add "Write a daily check-in note with priorities and blockers"
voxera daemon --once
voxera queue approvals list
# approve / deny / retry / cancel / pause / resume as needed

Path B

Dev VM setup (typed first-run)

1) Install + run setup

python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -e ".[dev]"

voxera setup

Writes config to ~/.config/voxera/config.yml; secrets go to keyring when possible (or a 0600 fallback file).

2) Validate provider endpoints

voxera doctor
voxera doctor --self-test

OpenRouter setup is supported out of the box with base URL, headers, and multi-tier brain roles (primary/fast/reasoning/fallback).

Queue Bootstrap

Required folders and first job

# Preferred one-time bootstrap
voxera queue init

# Equivalent manual fallback
mkdir -p ~/VoxeraOS/notes/queue/{inbox,pending/approvals,done,failed,artifacts,_archive}

# First queue job file
cat > ~/VoxeraOS/notes/queue/inbox/job-1.json <<'JSON'
{"version":"1","goal":"run a quick system check","mission_id":"system_check"}
JSON

systemctl --user restart voxera-daemon.service
voxera queue status
voxera queue approvals list

Operator Tip

Atomic producer write best practice

queue_dir=~/VoxeraOS/notes/queue
inbox_dir="$queue_dir/inbox"
mkdir -p "$inbox_dir"
job_id=job-$(date +%s)
tmp_path="$inbox_dir/.${job_id}.tmp"
final_path="$inbox_dir/${job_id}.json"
printf '{"goal":"run a quick system check"}
' > "$tmp_path"
mv "$tmp_path" "$final_path"
