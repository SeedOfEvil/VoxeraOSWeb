Quick Start · Alpha v0.1.5

Get Voxera OS running in minutes

Two paths: the streamlined Alpha baseline, or the full dev VM flow with typed setup, provider configuration, and queue initialization. All commands are copyable.

Path A — Recommended

Alpha baseline quick start

python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -e ".[dev]"

make update
make services-install

voxera --version
voxera queue status
voxera inbox add "Write a daily check-in note with priorities and blockers"
voxera daemon --once
voxera queue approvals list
# approve / deny / retry / cancel / pause / resume as needed

Path B

Dev VM setup with typed first-run

1 · Install and run setup

python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -e ".[dev]"

voxera setup

Writes config to ~/.config/voxera/config.yml. Secrets go to keyring when available, otherwise a 0600 fallback file.

2 · Validate provider endpoints

voxera doctor
voxera doctor --self-test

# quick offline check (no model calls)
voxera doctor --quick

OpenRouter is supported out of the box with base URL, headers, and multi-tier brain roles: primary, fast, reasoning, fallback.

3 · Try basic commands

voxera status
voxera skills list
voxera run system.status
voxera missions list

# plan with cloud AI
voxera missions plan "prep a focused work session" --dry-run

4 · Inspect config

voxera config show     # sensitive values redacted
voxera config validate # non-zero exit on errors
voxera config snapshot # write audit snapshot

Runtime config at ~/.config/voxera/config.json (optional). Precedence: CLI overrides → VOXERA_* env → config file → defaults.

Queue Bootstrap

Required folders and first job

# Preferred one-time bootstrap command
voxera queue init

# Equivalent manual fallback
mkdir -p ~/VoxeraOS/notes/queue/{inbox,pending/approvals,done,failed,artifacts,_archive}

# Drop your first queue job
cat > ~/VoxeraOS/notes/queue/inbox/job-1.json <<'JSON'
{"version":"1","goal":"run a quick system check","mission_id":"system_check"}
JSON

# Start or restart the daemon service
systemctl --user restart voxera-daemon.service

# Check status and resolve any approvals
voxera queue status
voxera queue approvals list
voxera queue approvals approve <job_id>

New in v0.1.5

Artifact hygiene commands

Prune stale artifacts

# Dry-run preview (safe, no changes)
voxera artifacts prune --max-age-days 30

# Actually delete
voxera artifacts prune --max-age-days 30 --yes

# Prune terminal queue buckets
voxera queue prune --max-age-days 30 --yes

Atomic producer write (best practice)

queue_dir=~/VoxeraOS/notes/queue
inbox_dir="$queue_dir/inbox"
mkdir -p "$inbox_dir"
job_id=job-$(date +%s)
tmp="$inbox_dir/.${job_id}.tmp"
final="$inbox_dir/${job_id}.json"
printf '{"goal":"run a quick system check"}\n' > "$tmp"
mv "$tmp" "$final"
