Docs Highlights

Operational contracts, sandbox model, and release guardrails

A concise documentation view of the Alpha v0.1.4 baseline, focusing on queue contracts, approvals, sandbox execution, and validation expectations before pushing changes.

Queue intake contract

  • Primary jobs: notes/queue/inbox/*.json only
  • Legacy root drops auto-relocate to inbox with queue_job_autorelocate
  • Mis-dropped pending primary jobs auto-relocate (never silently stuck)
  • Ready files only: daemon ignores dotfiles, *.tmp, *.partial

Approval lifecycle

  • Approval-required jobs move to pending/ (not failed)
  • Artifacts in pending/approvals/*.approval.json include policy reason + scope metadata
  • CLI resolves approvals: list / approve / approve --always / deny
  • Malformed approval artifact is surfaced as unparseable row and audited

Sandbox

Rootless Podman execution (MVP)

Install on Ubuntu 24.04

sudo apt update
sudo apt install -y podman uidmap slirp4netns fuse-overlayfs
podman info --debug | head

Security defaults

  • Default --network=none unless explicitly requested + approved
  • Read-only root filesystem
  • Writable mount limited to ~/.voxera/workspace/<job_id> as /work
  • Artifacts stored in ~/.voxera/artifacts/<job_id>/

Validation

Before pushing or merging

Required for PR merge

make merge-readiness-check

Runs quality checks + release consistency checks and aligns with the pre-push hook installed by make dev.

Broader local validation

make full-validation-check
# alias
make premerge

Includes merge-readiness plus failed-sidecar guardrails, full pytest, and E2E smoke.
