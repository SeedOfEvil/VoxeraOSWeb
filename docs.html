Operational contracts, sandbox model, and release guardrails

A focused documentation view of the Alpha v0.1.5 baseline — covering queue contracts, approval lifecycle, sandbox execution, and validation expectations before pushing changes.

Queue contracts

Intake, approvals, and lifecycle

Queue intake contract

  • Primary jobs: notes/queue/inbox/*.json — the only supported drop location.
  • Legacy root drops are auto-relocated to inbox with a queue_job_autorelocate audit event.
  • Mis-dropped pending primary jobs are also auto-relocated — never silently stuck.
  • Daemon ignores dotfiles, *.tmp, and *.partial artifacts.

Approval lifecycle

  • Approval-required jobs move to pending/ (never to failed).
  • Approval artifacts in pending/approvals/*.approval.json include policy reason and scope metadata.
  • CLI: list / approve / approve --always / deny.
  • Malformed approval artifacts are surfaced as "(unparseable)" rows and fully audited.

Failed-job sidecar contract

  • Sidecar path: failed/<job_stem>.error.json.
  • Required fields: schema_version (1), job, error, timestamp_ms.
  • Invalid sidecars are ignored in snapshots and logged as queue_failed_sidecar_invalid.
  • Schema evolution: reader uses an explicit supported-version allowlist; unknown versions are rejected.

Queue health & observability

  • Health counters persisted in notes/queue/health.json (shared by daemon and panel).
  • Brain fallback reasons classified: TIMEOUT, AUTH, RATE_LIMIT, MALFORMED, NETWORK, UNKNOWN.
  • voxera queue health shows paused flag, lock status, counters, last fallback, and last error summary.
  • voxera doctor --quick runs fast local checks with no model calls.

Sandbox

Rootless Podman execution

Install on Ubuntu 24.04

sudo apt update
sudo apt install -y podman uidmap slirp4netns fuse-overlayfs
podman info --debug | head

Security defaults

  • Default --network=none unless explicitly requested and approved.
  • Read-only root filesystem with --read-only.
  • Writable mount limited to ~/.voxera/workspace/<job_id> as /work.
  • Artifacts stored in ~/.voxera/artifacts/<job_id>/ (stdout, stderr, runner.json, command.txt).

Artifact pruning & queue hygiene

voxera artifacts prune

Deletes stale entries from notes/queue/artifacts/. Always dry-run unless --yes is passed.

# Preview only (safe, no changes)
voxera artifacts prune --max-age-days 30

# Delete entries older than 30 days
voxera artifacts prune --max-age-days 30 --yes

# Keep newest 50, prune the rest
voxera artifacts prune --max-count 50 --yes

# Machine-readable JSON summary
voxera artifacts prune --max-age-days 30 --json

voxera queue prune & reconcile

Remove stale job files from terminal buckets (done, failed, canceled). reconcile is report-only and never modifies state.

# Prune terminal job buckets
voxera queue prune --max-age-days 30 --yes

# Report-only hygiene diagnostic
voxera queue reconcile
voxera queue reconcile --json

Validation

Before pushing or merging

Required for PR merge

make merge-readiness-check

Runs formatting/lint checks, Mypy ratchet, and release consistency checks. Aligns with the pre-push hook installed by make dev.

Broader local validation

make full-validation-check
# alias
make premerge

Includes merge-readiness plus failed-sidecar guardrails, full pytest suite, and E2E smoke checks.
