Roadmap

Where Voxera OS is headed

Stability before scale. We build operator trust before adding capabilities. The alpha establishes the control structure that makes everything else safe to ship.

Now · Solid

Operator-grade baseline

  • Queue railguard: deterministic lifecycle, prune, reconcile
  • Human-in-the-loop approval gating + structured audit bundles
  • Daemon reliability: single-writer lock, graceful shutdown, startup recovery
Next · v0.1.6

Security + visibility hardening

  • Panel auth rate limiting + hardened endpoints
  • Ops health dashboard in panel
  • Health degradation signals + backoff
Later · v0.3

Voice stack

  • STT/TTS integration
  • Wake word detection: "Hey Voxera"
  • Voice-first command loop

Additional milestones

Beyond v0.1.6

v0.1.6 · near-term

Structured mission planning previews

Richer dry-run simulation with explicit operator confirmation pathways and clearer capability usage summaries before committing to a plan.

v0.2 · mid-term

Provider hardening + mission catalog expansion

Stronger OpenAI-compatible and Ollama provider behavior, improved fallback chain handling, and a mission catalog with 10+ built-in workflows.

v0.4 · later

Signed skills + marketplace

Signed skill bundles, a community skill marketplace, and ISO/image packaging for clean machine provisioning.

Alpha positioning

Trustworthy before ambitious

The alpha establishes the control structure — reliable queue, approval gating, audit artifacts, and safe daemon behavior — before we expand capabilities. Everything that comes after builds on this.
