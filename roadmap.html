Roadmap

Where VoxeraOS is headed

Stability before scale. Operator trust before expanding capabilities. VoxeraOS is early alpha — a one-person project growing into a product. Here's what's built, what's next, and where we're going.

Current alpha

Built in v0.1.7

Built

Governed queue execution

  • Durable filesystem-backed queue with explicit job lifecycle (inbox → pending → running → done)
  • Approval gates that pause execution — jobs wait until operator explicitly allows them
  • Artifact-backed outcomes — every job produces evidence of what requested, allowed, executed, and resulted
Built

Security, visibility, and baseline recovery

  • Input guardrails, prompt boundaries, and rate-limited panel access protection
  • Operator panel: health at a glance, hygiene tools, last run results
  • Baseline recovery directories, quarantine buckets, and incident bundles operational
Built

Vera / VoxeraOS separation and reliability

  • Clear separation between Vera (conversational surface) and VoxeraOS (governed execution runtime)
  • Smoother, more predictable approval flow — reduced friction, no dead-ends
  • Deterministic prompt ordering, brain fallback classification, and consistent input handling

Immediate next milestone

Next up: Recovery Inspector UI

Next up

Richer recovery and quarantine inspection

  • Baseline recovery directories, quarantine buckets, and incident bundles are already in v0.1.7
  • Next milestone: dedicated panel view for inspecting recovery sessions and quarantined items visually
  • Download incident bundles directly from the panel — richer triage and visualization tooling

Near-term

Coming after: Long-run resilience

Coming

Degradation awareness

  • Runtime becomes self-aware of sustained failures
  • Surfaces degradation state clearly to operators
  • No silent degradation — you always know the system's true state
Coming

Intelligent backoff

  • Automatic backoff when providers are failing or rate-limiting
  • Clear shutdown and exit context in health surfaces
  • Graceful handling of extended provider outages

Future

  • Advanced rate limiting and usage controls
Future

Voice and marketplace

  • Voice loop expansion — "Hey Voxera" wake word
  • Signed skills marketplace for community contributions
  • Release and packaging improvements

Our philosophy

Grounded in what AI is missing today

This project is grounded in present-day gaps in AI systems: trust, review, execution boundaries, and proof of outcome. The alpha establishes the control structure — governed execution, approval gating, audit records, and safe defaults — before expanding capabilities. Open-source public release is coming soon.
